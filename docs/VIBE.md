# Wakey — The Vibe

## What Makes Something Go Viral

OpenClaw didn't go viral because of architecture. It went viral because:
1. People SAW it and immediately wanted it
2. 30-second demo was enough to understand the value
3. It solved a real pain point beautifully
4. Community could contribute and extend it

## Inspiration

### Tabbie (Physical)
- AMOLED expressions that react to your actions
- Celebrates task completion
- Breathes when idle
- Physical presence = emotional connection

### Vibe Island (Software)
- Pixel art that BREATHES — ASCII characters flicker and regenerate
- Mouse proximity triggers glow effects
- CRT retro-futuristic aesthetic
- 8-bit synthesized sound alerts
- The UI itself feels alive — not just the content

## Wakey Must Feel Like This

Not a window. Not a widget. A CREATURE on your screen.

### The Alive Loop (Core Experience)

```
IDLE:     Sprite breathes slowly, ambient glow pulses with heartbeat tick
          Eyes occasionally blink, look around
          Subtle particle effects (floating dots, gentle shimmer)

AWARE:    When you switch apps, sprite looks toward the change
          Eyes track cursor movement (subtle, not creepy)
          Glow color shifts based on what app you're in
          (Code editor = soft blue, browser = warm orange, terminal = green)

THINKING: When LLM is processing, sprite shows thinking animation
          Particles accelerate, glow intensifies
          Small loading animation that feels organic, not mechanical

SPEAKING: Chat bubble appears with typewriter effect
          Sprite's expression matches emotion (happy, concerned, playful)
          Subtle sound cue (optional, 8-bit style like Vibe Island)
          Bubble has soft glow, rounded edges, feels warm

REACTING: Task completed → mini celebration (confetti particles, happy bounce)
          Build failed → concerned look, red glow flicker
          Long idle → sleepy animation, dim glow
          User returns after idle → wake-up animation, stretch, greeting

FLOW:     When user is in deep focus, Wakey goes minimal
          Almost invisible, just a faint breathing glow
          No interruptions, no bubbles
          Presence without intrusion
```

### Visual Identity

The character must be:
- Simple enough to animate smoothly at 60fps
- Expressive enough to convey emotion
- Customizable (users pick their own companion style)
- Memorable (instantly recognizable, meme-able)

Inspiration styles:
- Pixel art creature (like Tamagotchi, but modern)
- Geometric blob with eyes (like a slime from games)
- Minimalist ghost/spirit (cute, floaty)
- Abstract glow orb with personality

### Sound Design (Optional, Off by Default)
- Soft chime when Wakey speaks
- 8-bit celebration on task completion
- Gentle ambient hum when in flow
- All sounds must be: short, non-intrusive, satisfying

### The 30-Second Demo That Goes Viral

1. User opens terminal, runs `wakey`
2. A cute creature appears in the corner, breathes, blinks
3. User opens VS Code — creature's glow shifts blue, eyes look toward the editor
4. User starts coding — creature goes quiet, just breathing
5. User gets stuck (same file for 3 min) — creature gently says "Want help with that?"
6. User says yes — creature reads the code, offers a suggestion
7. User fixes the bug — creature does a little celebration dance

That's the video. That's the tweet. That's the movement.

## Community & Growth Strategy

### Open Source First
- MIT license, fully open
- Easy to contribute (well-documented crates)
- Skill marketplace (community-built extensions)

### Character Ecosystem
- Users create and share custom companion designs
- Sprite sheets are just PNG files — easy to create
- Community votes on featured companions
- "Adopt a Wakey" — pick from gallery

### Auto-Farming / Passive Growth
- Wakey learns from you → gets smarter → you tell friends
- Skill sharing → "My Wakey learned to deploy, here's the skill file"
- Screenshot-worthy moments → organic social sharing
- Integration with popular tools → natural discovery
