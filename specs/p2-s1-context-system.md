# P2-S1: Context System (wakey-context)

## Goal
Filesystem-backed memory + context with SQLite index. OpenViking L0/L1/L2 tiers. Foundation for skills and agent supervision.

## Architecture

```
~/.wakey/
├── context/                    # Filesystem = source of truth
│   ├── user/
│   │   └── memories/           # User preferences, patterns
│   ├── agent/
│   │   ├── skills/             # SKILL.md files live here
│   │   └── memories/           # Agent's learned patterns
│   ├── session/                # Current session working memory
│   └── resources/              # External knowledge (docs, repos)
└── index.db                    # SQLite FTS5 index (rebuilt from filesystem)
```

## What to implement

### 1. Filesystem layer (src/filesystem.rs)
- `ContextPath` struct — represents `wakey://user/memories/preferences`
- `ContextFs::read(path)` → String
- `ContextFs::write(path, content)`  
- `ContextFs::list(path)` → Vec<ContextEntry>
- `ContextFs::delete(path)`
- `ContextFs::exists(path)` → bool
- Base dir: `~/.wakey/context/` (from config)
- Create preset directories on first run (user/memories, agent/skills, agent/memories, session, resources)

### 2. Tiered loading (src/tiers.rs) — OpenViking pattern
- Every directory gets auto-generated:
  - L0: one-line summary (~100 chars) — stored in index.db
  - L1: paragraph overview (~500 chars) — stored in index.db  
  - L2: full content — read from filesystem on demand
- `Tiers::get_l0(path)` → String (from index)
- `Tiers::get_l1(path)` → String (from index)
- `Tiers::get_l2(path)` → String (reads file)
- L0/L1 generated by LLM on first index, cached in SQLite
- Regenerate when file modified (check mtime)

### 3. SQLite index (src/memory.rs) — ZeroClaw Memory trait
```rust
#[async_trait]
pub trait Memory: Send + Sync {
    async fn store(&self, key: &str, content: &str, category: &str) -> WakeyResult<()>;
    async fn recall(&self, query: &str, limit: usize) -> WakeyResult<Vec<MemoryEntry>>;
    async fn forget(&self, key: &str) -> WakeyResult<bool>;
    async fn list(&self, category: Option<&str>) -> WakeyResult<Vec<MemoryEntry>>;
}
```

SQLite schema:
```sql
-- Content index with FTS5
CREATE VIRTUAL TABLE context_fts USING fts5(
    path, title, content, category, l0_abstract, l1_overview
);

-- Metadata
CREATE TABLE context_meta (
    path TEXT PRIMARY KEY,
    category TEXT NOT NULL,
    mtime_secs INTEGER NOT NULL,
    size_bytes INTEGER NOT NULL,
    l0_abstract TEXT DEFAULT '',
    l1_overview TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);

-- Quality metrics (for skills — OpenSpace pattern)
CREATE TABLE skill_metrics (
    skill_id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    total_selections INTEGER DEFAULT 0,
    total_applied INTEGER DEFAULT 0,
    total_completions INTEGER DEFAULT 0,
    total_fallbacks INTEGER DEFAULT 0,
    last_used TEXT,
    created_at TEXT NOT NULL
);

-- Skill lineage (OpenSpace version DAG)
CREATE TABLE skill_lineage (
    skill_id TEXT NOT NULL,
    parent_skill_id TEXT NOT NULL,
    evolution_type TEXT NOT NULL,  -- 'fix', 'derived', 'captured'
    change_summary TEXT DEFAULT '',
    PRIMARY KEY (skill_id, parent_skill_id)
);
```

### 4. Retrieval (src/retrieval.rs) — OpenViking directory-recursive
- `retrieve(query, limit)` → Vec<ContextEntry>
- Step 1: FTS5 search across l0_abstract + l1_overview
- Step 2: BM25 ranking
- Step 3: Return top-N with L0 summaries
- Step 4: Caller can request L2 (full content) for specific results
- Token-efficient: broad search uses L0/L1 only, deep read uses L2
